Overview

This site is driven by a single JSON dataset (nodes, edges, timeline, and sources). It’s designed for evidence-first navigation.

confirmed alleged disputed ✅ Official 🧾 Testimony 📰 Media

Data file: /data/latest.json (fallback). If you add an API function, /api/dataset will override the static file.

How updates work

  1. Commit an updated data/latest.json (simplest), or
  2. Deploy an Azure Function at /api/dataset to serve JSON from your pipeline.

Tip for accuracy

Treat non-official sources as “reported” and label edges as alleged/disputed until confirmed by official documents.

Dataset schema

{
  "meta": { "asOf": "ISO-8601 string" },
  "nodes": [
    {"id":"node_id","kind":"person|org|event|evidence","label":"...","subtitle":"...","tags":["..."]}
  ],
  "edges": [
    {"id":"edge_id","from":"node_id","to":"node_id","type":"relationship_type",
     "strength":"official|testimony|media","status":"confirmed|alleged|disputed",
     "evidenceIds":["source_id"],"explanation":"..."}
  ],
  "evidence": [
    {"id":"source_id","title":"...","tier":"official|testimony|media","date":"YYYY-MM-DD",
     "publisher":"...","url":"https://...","tags":["..."],"summary":"..."}
  ],
  "timeline": [
    {"id":"tl_id","date":"YYYY-MM-DD","tier":"official|testimony|media","title":"...",
     "summary":"...","sourceIds":["source_id"],"tags":["..."]}
  ]
}
